Gemini CLI Commands Reference Guide

A complete breakdown of terminal flags, interactive slash commands, and input triggers.

1. Interactive Slash Commands (Inside CLI Session)

These commands are executed inside the live interactive REPL session to control configuration, memory, tools, and conversation states.

Session & Chat Management

Command Purpose & Explanation
/help Displays comprehensive help text and lists all currently available environment commands.
/clear Clears the visual terminal interface screen and resets the model's active context window back to empty.
/quit or /exit Gracefully terminates the live interactive AI agent session and returns control back to the system shell.
/chat save <tag> Saves the current state of the chat thread history under a unique user-specified tag name.
/chat resume <tag> Reloads and populates a previously saved conversation thread context back into the live window.
/chat list Scans the local active workspace environment directory and prints all stored session checkpoint tags.
/chat delete <tag> Permanently erases a specific conversation history checkpoint tag from the disk storage.
/chat share [file.md/.json] Exports the complete dialogue log directly into a cleanly formatted external Markdown or structural JSON file.

Environment & Tool Integration

Command Purpose & Explanation
/tools Lists every function call tool available to the model. Use /tools desc to view extended capabilities parameters.
/mcp Manages Model Context Protocol servers. Subcommands like list or reload connect external enterprise data infrastructure dynamically.
/directory add <path> Appends a specified localized directory pathway contextually into the AI agent's searchable workspace footprint.
/directory show Renders a tree-like view of all system folder frameworks currently exposed to the model context.
/memory add <text> Stores long-term custom system prompt constraints or localized developer rule configurations across continuous runs.
/memory show Outputs all registered developer persona parameters, coding guardrails, and styling instructions stored in memory.
/model Triggers an interactive interface toggle switch to switch models smoothly (e.g., Flash for fast responses, Pro for deep analytics).

2. CLI Core Startup Flags (Terminal Launching)

These arguments adjust the initialization configurations of the tool when called directly from standard system prompt scripts.

Flag Option Purpose & Explanation
gemini -p "query"
--prompt "query"		 Executes a strict non-interactive command operation. The model interprets the prompt, renders standard output, and exits immediately.
gemini -i "query"
--prompt-interactive		 Starts the CLI by evaluating the initial text prompt input and seamlessly drops you straight into the live conversation prompt.
gemini -m "<name>"
--model "<name>"		 Explicitly overrides the baseline system config file model choices with a specified model version.
gemini -r "latest"
--resume		 Automatically looks up and restores the last active workspace session thread, minimizing context loss.
gemini --approval-mode Controls agent shell operation execution safety boundaries: default (always prompts), auto_edit (auto-edits, asks on runs), or yolo (no prompts).
gemini --include-directories Pre-loads designated lists of separated workspace structural pathways straight into context on launch.

3. Special Input Triggers

Prefix notation shortkeys used to signal complex ingestion context parsing right inside the active terminal chat line.

Prefix Symbol Purpose & Explanation
@<file_path> Direct Context Injection: Instructs the tool to pull the entire contents of a specific file or directory directly into the context window.
!<command> Direct Shell Escape: Temporarily passes the instruction directly to the system's local shell terminal environment without closing the session.
